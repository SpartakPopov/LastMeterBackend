# CI/CD, Docker, and SonarQube setup

All code changes are made on `ci-cd-docker-sonarqube`. Do not merge it until
the GitHub variables, secrets, SonarQube projects, and deployment server below
are ready.

## 1. Prepare the deployment server

Use a Linux server with Docker Engine, the Docker Compose plugin, SSH access,
and ports 22 and 80 open.

Create a dedicated deployment user and allow it to run Docker:

```bash
sudo useradd --create-home --shell /bin/bash deploy
sudo usermod -aG docker deploy
sudo mkdir -p /opt/lastmeter
sudo chown deploy:deploy /opt/lastmeter
```

Add the public half of a dedicated SSH key to
`/home/deploy/.ssh/authorized_keys`. Store the private half in GitHub as
`DEPLOY_SSH_KEY`.

## 2. Configure GitHub for the backend repository

Open `Settings -> Secrets and variables -> Actions`.

Repository variables:

| Name | Example |
| --- | --- |
| `DEPLOY_HOST` | `203.0.113.10` |
| `DEPLOY_PORT` | `22` |
| `DEPLOY_USER` | `deploy` |
| `DEPLOY_PATH` | `/opt/lastmeter` |
| `GHCR_USERNAME` | GitHub username that can read both images |
| `SONAR_HOST_URL` | `https://sonarqube.example.com` |
| `SONAR_PROJECT_KEY` | `lastmeter-backend` |

Repository secrets:

| Name | Purpose |
| --- | --- |
| `DEPLOY_SSH_KEY` | Private SSH key for the deployment user |
| `GHCR_TOKEN` | Fine-grained token that can read both GHCR packages |
| `DB_PASSWORD` | Production MySQL root/application password |
| `SONAR_TOKEN` | Token generated by the backend SonarQube project |

Create a GitHub environment named `production`. Optional environment approval
rules can be enabled before deployment.

## 3. Configure GitHub for the frontend repository

Create the same deployment variables and secrets, except `DB_PASSWORD`.
Use a separate SonarQube project and set `SONAR_PROJECT_KEY` to
`lastmeter-frontend`.

## 4. Start SonarQube Community

SonarQube can run on a separate server with the provided Compose file:

```bash
export SONAR_DB_PASSWORD='replace-with-a-strong-password'
docker compose -f deployment/sonarqube-compose.yml up -d
```

Open `http://SERVER:9000`, change the default administrator password, create
one project for each repository, and generate a project token for each.
The SonarQube URL must be reachable from GitHub-hosted runners.

## 5. First deployment

Merge and deploy the backend repository first. Its workflow uploads the
production Compose file and starts MySQL, backend, and frontend. Subsequent
frontend merges update only the frontend service.

The application is served on `http://DEPLOY_HOST`. The frontend Nginx
container proxies API requests to the backend container on the private Docker
network.
